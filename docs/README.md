# Documentation

Project documentation organized by status.

## Structure

```
docs/
├── active/      # Current plans and tasks in progress
├── archive/     # Completed work and historical records
│   └── reports/ # Past analysis reports
└── ideas/       # Future concepts and brainstorming (not approved)
```

## Directories

### `/active` - Present
Current implementation plans, active tasks, and work in progress.

| File | Description |
|------|-------------|
| `ux-improvement-plan.md` | Portfolio UX overhaul - Phase 1 complete |

### `/archive` - Past
Completed plans, historical analysis, and reference material.

| File | Description |
|------|-------------|
| `anglerfish-scrollytelling-recommendations.md` | Animation recommendations (implemented) |
| `reports/` | June 2025 portfolio analysis reports |

### `/ideas` - Future
Draft concepts, brainstorming, and proposals not yet approved for implementation.

| File | Description |
|------|-------------|
| `portfolio-strategy.md` | Strategic positioning framework |
| `portfolio-copy.md` | Draft copy/content ideas |
| `single-page-structure.md` | Page structure proposals |

## Workflow

1. **New ideas** → Start in `/ideas`
2. **Approved for work** → Move to `/active`
3. **Completed** → Move to `/archive`
