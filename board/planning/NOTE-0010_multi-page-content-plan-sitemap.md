# Multi-Page Content Plan & Sitemap

## Why This Is High Priority

The site is transitioning away from a single-page structure — content that currently lives on the home page is being broken out into dedicated sub-pages. **Before any sub-page work begins, we need a finalized sitemap and content plan** so that:

- Each sub-page has a clear purpose and scope before it gets built
- Home page can be redesigned with confidence about what stays vs. what moves
- Navigation, routing, and information architecture decisions are made once, not improvised per page
- We avoid shipping half-migrated content or duplicated sections across home + sub-pages

This is a **blocking prerequisite** for all sub-page creation work.

## What This Note Should Become

A concrete plan that answers:

1. **Full sitemap** — every page/route on the new multi-page site, with a one-line purpose for each
2. **Content inventory** — what currently lives on the home page, and for each item: stays on home / moves to sub-page X / gets cut / gets rewritten
3. **Home page scope** — what the new trimmed-down home page is *for* (hero + teasers + CTAs? landing + filtered highlights?)
4. **Navigation model** — how users move between home and sub-pages (primary nav, breadcrumbs, footer links, in-content cross-links)
5. **URL structure** — clean routes, slug conventions, whether sections get their own directories
6. **Migration order** — which sub-page to build first (likely the one that unblocks the most home-page cleanup)
